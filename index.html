    Contrive

    Contrive is a shard similar for devise with rails. Contrive handles everything authentication. Currently amber is the only supported framework with both granite and jennifer orms being supported. I am open to other frameworks being supported but there is a lot to do before that. If you want this to work with lucky, you'll have to open a pull request right now.

    Contrive takes the same approach as devise which is modularity. No need to load up code you aren't planning on using. The CLI generator makes installation easy. Please give a thanks to the amber core team for their CLI. I have...shamelessly...ripped it out of amber for contrive. Contrive CLI probably wouldn't even be possible if it weren't for the amber team.

    Note: Only the class 'User' is supported right now. This will change in future releases.

    Currently, contrive has 3 modules:

    1. Authentication
    • Just like amber or lucky, this is basic authentication. Sign up, sign in, sign out.
    • Mandatory columns:
    • email & password_digest > Note: Contrive also puts 'password' and 'new_password' in as attributes, however we don't want these saved in plain text so do not add them as columns.
    1. Confirmable
    • Users are required to confirm their email prior to activation. Contrive generates a UUID for each user and sends an email.
    • Mandatory columns:
    • token, confirmed & confirmed_at.
    1. Omniauth
    • Users can sign up with google, facebook, github, twitter or vk.
    • Manadatory columns:
    • uid

    Installation

    1. Add the dependency to your shard.yml:

    `yaml dependencies:

     contrive:
   github: andrewc910/contrive

    `

    1. Run shards install
    1. Move onto usage.

    Usage

    Authentication

    1. Run the generator in console depending on your ORM: 
    **Granite:**  
```bash
contrive g auth user granite
```  

or  

**Jennifer:**  
```bash
contrive g auth user jennifer
```
    1. Migrate: 
    **Granite:**  
```bash
amber db migrate
```

or

**Jennifer:**  
```bash
crystal sam.cr -- db:migrate
```

    Confirmation

    1. Run the generator in console depending on your ORM: 
    **Granite:**  
```bash
contrive g confirm user granite
```  

or

**Jennifer:**  
```bash
contrive g confirm user jennifer
```
    1. Migrate
    1. Uncomment the commented lines in your user model that talk about confirmable.
    1. Open contrive.cr in initializers. Change config.confirmable_enabled to true.
    1. Configure SMTP - Please follow amber guides for this.
    Note: Amber will print out the token in console even if smtp is disabled. Find it, and paste with your_url/registration/confrim?token= to activate an account if you don't want to configure smtp on local.

    Omniauth

    1. Run the generator in console depending on your ORM: 
    **Granite:**  
```bash
contrive g omni user granite
```

or  

**Jennifer:**  
```bash
contrive g omni user jennifer
```
    1. Migrate
    1. Open up contrive.cr in initializers
    • Configure you auth providers app id and secret key
    • Uncomment the 'requires' for any providers you need
    1. Add Sign in with Github to a view and test. Replace github in the href for any providers you configured.
    1. Uncomment the commented line in pipes/authenticate.cr
    1. Add include_providers() to your authenticable model along with the providers names you required in step 3. EX: include_providers(:facebook, :google)

    Roadmap

    • Tests: Nothing is stable without comprehensive tests and a beta release will not happen until this is done.
    • CLI: A lot of values are hardcoded. Fixing this will open up the door to other classes being supported.
    • I am sure more will come
